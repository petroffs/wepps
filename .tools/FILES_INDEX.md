## 🧪 Тестовые файлы Bruno

### Папка: `.tools/bruno/WeppPlatformV1/clientAPP/goods/`

Все файлы в этой папке содержат полный набор тестов уровней L1-L5.

| Файл | Метод | Параметры | Тесты | Статус |
|------|-------|-----------|-------|--------|
| `goods.get.full-tests.bru` | GET /v1/goods | page, limit, category, filters, sort, search | 34 тестов | ✅ |
| `goods.item.full-tests.bru` | GET /v1/goods.item | id (required) | 28 тестов | ✅ |
| `goods.categories.full-tests.bru` | GET /v1/goods.categories | нет | 22 тестов | ✅ |
| `goods.filters.full-tests.bru` | GET /v1/goods.filters | category, search | 32 тестов | ✅ |
| `goods.favorites.full-tests.bru` | GET /v1/goods.favorites | авторизация | 26 тестов | ✅ |
| `goods.variations.full-tests.bru` | GET /v1/goods.variations | goodsId (required), page, limit | 36 тестов | ✅ |

**Использование:**
```bash
# Запустить один тест
bruno run goods.get.full-tests.bru

# Запустить всю папку
bruno run goods/ --output-format json

# С конкретным окружением
bruno run goods/ --env production
```

## 🚀 Как начать разработку новго метода

### Шаблон: Разработка нового метода `GET /v1/orders`

**Шаг 1: Спецификация (15 мин)**
```
Прочитаю раздел "Применение к методам" в TESTING_METHODOLOGY.md
Определю параметры, ответ, коды ошибок
```

**Шаг 2: Базовые тесты (30 мин)**
```
Копирую goods.get.full-tests.bru как шаблон
Адаптирую L1-L3 тесты под orders.get
Запускаю: bruno run orders.get.full-tests.bru
```

**Шаг 3: Реализация (зависит от сложности)**
```
Пишу метод getOrders() в RestV1APP.php
Использую guard clauses для ошибок
Возвращаю {status, message, data, pagination}
```

**Шаг 4: Логические тесты (30 мин)**
```
Добавляю L4-L5 тесты в файл
Запускаю: bruno run orders.get.full-tests.bru
Исправляю ошибки в тестах или коде
```

**Шаг 5: Финальный review (15 мин)**
```
Все тесты проходят ✅
Документация полная ✅
Code review пройден ✅
```
